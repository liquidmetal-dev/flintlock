# AGENTS.md

Guidance for AI coding agents working in this repository. Humans should read
[`CONTRIBUTING.md`](CONTRIBUTING.md) — it is the full, authoritative guide. This
file is the agent-facing digest and defers to it on any conflict.

## Project overview

`flintlock` is a Go service for creating and managing [Firecracker](https://firecracker-microvm.github.io/)
microVMs over a gRPC API. The daemon is `flintlockd`.

Key directories:

- `cmd/` — binary entrypoints (`flintlockd`, metrics)
- `core/` — domain logic (ports, application services)
- `internal/`, `pkg/` — implementation packages
- `api/` — protobuf/gRPC definitions
- `client/` — generated/consumer client code
- `test/` — e2e and integration tests
- `userdocs/` — documentation site (npm)
- `hack/` — scripts and build tooling

## Build, test, lint

Run from the repo root:

- `make build` — build the binaries
- `make test` — run unit tests
- `make lint` — run golangci-lint (config in `.golangci.yml`)
- `make generate` — regenerate mocks, protobuf, and DI wiring; run this after
  changing anything under `api/` or any mocked interface
- `make test-e2e` — e2e tests (require KVM + Firecracker; normally CI only)

Every commit must be compilable with `make test` passing.

## Commit & PR rules

- **Keep changes small.** One focused concern per PR and per commit. Split
  unrelated work into separate PRs rather than bundling it.
- **No `Co-Authored-By:` trailers** in commit messages.
- **No agent or AI names** (e.g. Claude, Codex, "generated by …") anywhere in
  commit messages, commit titles, or PR titles.
- **Commit message format** (see CONTRIBUTING.md § *Commit message formatting*):
  - Subject ≤ 70 chars describing *what* changed.
  - Blank line, then a body wrapped at ~80 chars describing *why* and *how*.
- **Sign off** every commit for DCO (`Signed-off-by:`, i.e. `git commit -s`).
- Squash noise commits (`fix stuff`, `wip`) before requesting review.
- **PR title** follows the same rule as a commit subject — it lands in the
  release notes, so make it clear and concise.
